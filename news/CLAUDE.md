# news/CLAUDE.md

이 문서는 `news` 폴더 안의 파일을 다룰 때만 적용되는 세부 규칙입니다. 프로젝트 전체 공통 규칙(보안, 실패 허용 원칙 등)은 루트 `CLAUDE.md`를 따르며 여기서는 반복하지 않습니다.

## 담당 범위

이 폴더는 알람 앱의 **'최근뉴스' 조회 및 TTS(음성 읽어주기) 브리핑 기능**만 담당한다. 알람 자체(시간 설정, 알람 소리 재생/중지 등)의 로직은 이 폴더의 책임이 아니다.

## 파일 구조 규칙

- 이 폴더에는 `news.js`, `news.css` 두 파일만 둔다.
- 새로운 파일을 추가하고 싶어지더라도 이 두 파일 안에 구현하는 것을 우선 고려하고, 구조 변경이 꼭 필요하다고 판단되면 임의로 파일을 늘리지 말고 먼저 사용자에게 확인한다.

## 함수 이름 규칙

- 데이터를 가져오는 함수: `fetchXxx` (예: `fetchNews`)
- 화면에 그리는(렌더링) 함수: `renderXxx` (예: `renderNewsCards`)
- 순수 변환 함수(입력을 받아 가공된 값을 반환하고 부수효과가 없는 함수): `buildXxx` / `stripXxx` (예: `buildNewsCardHtml`, `stripHtmlTags`)
- 이 네이밍 규칙에 맞지 않는 새 함수를 만들 때는 위 접두사 중 가장 가까운 것을 따르되, 애매하면 사용자에게 확인한다.

## 함수 시그니처

- 함수 시그니처(파라미터 구성 등)는 `최근뉴스_기술명세서.md`와 `prd.md`에 명시되거나 합의된 흐름을 그대로 따른다.
- 이미 정해진 파라미터 이름이나 순서를 임의로 바꾸지 않는다. 명세에 없는 시그니처를 새로 정해야 하는 경우, 임의로 결정하지 말고 사용자에게 먼저 확인한다.

## script.js와의 관계

- `script.js`(알람 로직)는 이 폴더가 제공하는 함수(`fetchXxx`, `renderXxx` 등)를 **호출만** 한다.
- 뉴스 검색, 뉴스 데이터 가공, TTS 재생 등 뉴스 관련 로직을 `script.js`에 직접 작성하지 않는다. 관련 로직은 항상 `news.js`에 작성하고, `script.js`에서는 필요한 시점(예: 알람 끄기 클릭 이벤트)에 `news.js`의 함수를 호출하는 코드만 둔다.
